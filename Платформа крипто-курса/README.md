# Тесты курса BlockCapital Crypto Summit

Статическая веб-платформа для тестирования по занятиям курса (16 занятий).

Что умеет:
- отдельный тест для каждого занятия;
- автопроверка закрытых вопросов (один вариант / множественный выбор / сопоставление / порядок шагов);
- сохранение прогресса и результатов в браузере студента (localStorage);
- работает без backend, разворачивается как статика.

Сейчас открыты тесты **занятий 1 и 2**. Остальные занятия видны в списке как «Скоро» и
открываются по мере готовности тестов.

## Запуск локально

```bash
# 1. собрать данные из JSON-файлов занятий
node scripts/build-course-data.mjs

# 2. поднять локальный сервер
python3 -m http.server 4185

# 3. открыть в браузере
#    http://localhost:4185
```

## Откуда берутся данные

Источник истины — **верифицированные JSON-файлы по одному на занятие**:

- `data/lessons/lesson-01.json`
- `data/lessons/lesson-02.json`
- … (lesson-NN.json по мере готовности)

Сборщик `scripts/build-course-data.mjs` валидирует их и склеивает в `data/course-data.json`,
который читает фронтенд.

> Почему JSON, а не парсинг markdown, как в AI-хабе: готовые тесты пайплайна выходят в разном
> формате от занятия к занятию (заголовки `## Q1 —` vs `## Q1.`, метаданные в скобках vs
> буллеты, сопоставления двумя таблицами vs одной, мультивыбор инлайн-галочками ☑/☐).
> Жёсткий парсер на этом ломается, поэтому каждое занятие конвертируется в чистый JSON один раз
> и проверяется глазами.

## Как добавить тест нового занятия

1. Возьмите готовый `Занятие_NN_Тест_готовый.md` из папки занятия.
2. Сконвертируйте его в `data/lessons/lesson-NN.json` по образцу `lesson-01.json` / `lesson-02.json`.
   Поддерживаемые `interaction`: `single_choice`, `multi_choice`, `matching_text`, `ordering`, `open_text`.
3. Откройте занятие на фронтенде: в `app/main.js` добавьте его id в `AVAILABLE_MODULE_IDS`
   (список `DISPLAY_MODULES` уже содержит все 16 занятий).
4. Пересоберите данные: `node scripts/build-course-data.mjs`.
5. Проверьте локально и запушьте.

### Схема вопроса в JSON

```jsonc
{
  "number": 1,
  "type": "Сопоставление",          // ярлык типа (показывается студенту)
  "goal": "З-1",                     // цель Блума (необязательно)
  "lesson": "Т3, Т6",               // блок занятия (необязательно)
  "difficulty": "Лёгкий",
  "scoring": "Все 6 пар верно.",    // критерий зачёта (показывается в фидбэке)
  "interaction": "matching_text",   // тип взаимодействия
  "promptMarkdown": "…",            // текст вопроса (markdown). Для matching — нумерованный
                                     //   список «левого столбца» под заголовком ###
  "options": [                       // single/multi — варианты; matching — правый столбец
    { "key": "A", "label": "…" }
  ],
  "correctAnswer": "…",             // человекочитаемый правильный ответ (фидбэк)
  "explanation": "…",               // пояснение (фидбэк)
  "ordering": null,                  // только для interaction=ordering: { items[], solution[] }
  "grading": {                       // машинная проверка
    "mode": "matching_text",
    "expectedMap": { "1": "A", "2": "B" },  // номер левого → буква правого
    "minCorrect": 6
  }
}
```

Типы `grading`:
- `single_choice` → `{ "mode": "single_choice", "correctKey": "B" }`
- `multi_choice` → `{ "mode": "multi_choice", "correctKeys": ["A","B"] }` (засчитывается **точное** совпадение набора)
- `matching_text` → `{ "mode": "matching_text", "expectedMap": {…}, "minCorrect": N }`
- `ordering` → `{ "mode": "ordering", "solution": [4,2,1,3] }` (для items[i] — правильная позиция 1..n)

## Публикация на GitHub Pages

В репозитории есть workflow `.github/workflows/deploy-tests-pages.yml`. Он на каждый push в `main`
пересобирает `course-data.json` и публикует папку `Платформа крипто-курса` в Pages.

Один раз на GitHub:
1. Залить проект в репозиторий.
2. `Settings → Pages → Build and deployment → Source: GitHub Actions`.
3. Убедиться, что основная ветка — `main`.

## Важно

- Тест — образовательный контент, **не инвестиционная рекомендация**.
- Тесты проверяют понимание и механику, а не запоминание volatile-фактов (курсы, APY, комиссии, даты).
- Результаты хранятся **только в браузере студента**. Для централизованного сбора нужен внешний
  приёмник (Google Sheets через Apps Script, Supabase, Firebase) — это следующий шаг.
